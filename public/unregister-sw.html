🔧 Service Worker Management

This page helps you unregister the service worker during development/testing.

Checking service worker status...

📋 Why Unregister?

  • Development/Testing: Service workers cache files, making it hard to see updates
  • Debugging: Old cached code can cause confusing bugs
  • Fresh Start: Ensures you're always seeing the latest version

🔄 After Unregistering:

  1. Service worker will be removed from your browser
  2. All caches will be cleared
  3. App will work normally, just without offline support
  4. Hard refresh (Ctrl+Shift+R / Cmd+Shift+R) to ensure clean state

✅ When Ready to Launch:

To re-enable the service worker before launch:

  1. Open sw.js
  2. Change const SW_ENABLED = false; to const SW_ENABLED = true;
  3. Deploy to production
  4. Service worker will automatically register again

Manual Unregistration (if needed):

  1. Open browser DevTools (F12)
  2. Go to Application tab (Chrome) or Storage tab (Firefox)
  3. Click Service Workers in the left sidebar
  4. Click Unregister next to each service worker
  5. Go to Cache Storage and delete all caches
  6. Refresh the page
